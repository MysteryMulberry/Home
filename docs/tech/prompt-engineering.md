# 提示词工程实践指南

## 核心原则

### 1. 明确指令
```
请将以下英文文本翻译为中文，保持专业术语的准确性，
并使用Markdown格式输出：

[文本内容]
```

### 2. 提供上下文
```
你是一位资深的Python开发者，专注于数据科学领域。
请审查以下代码，指出性能问题和改进建议：

[代码]
```

### 3. 分步推理 (Chain of Thought)
```
请逐步分析以下问题：
1. 首先，识别输入数据的关键特征
2. 然后，选择合适的算法
3. 最后，给出具体实现

问题：[具体问题]
```

## 高级技术

### Few-Shot Learning
```
以下是几个示例：
输入: "hello" → 输出: "HELLO"
输入: "world" → 输出: "WORLD"
输入: "python" → 输出: ?
```

### 角色设定
```
你是一个代码审查专家，有以下专长：
- 识别安全漏洞
- 优化算法复杂度
- 改善代码可读性
```

### 输出格式控制
```
请以JSON格式返回结果，包含以下字段：
- "summary": 摘要（不超过50字）
- "score": 评分（1-10）
- "suggestions": 改进建议列表
```

## 常见陷阱
- 指令模糊导致输出不一致
- 过长上下文导致模型遗忘
- 未验证输出格式
- 忽略模型的能力边界
